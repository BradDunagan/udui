Below is an SVG element with margin and padding set at 4 pixels (you can change that with the F12 debugger). In that is a root <g> that is drawn with JavaScript along with its contents. Its border is set to be large (~9 pixels), black and semitransparent - its width is called the containerBorderWidth. The width of the root <g> is determined from the inside width of the parent <SVG> element with <node>.width.baseVal.value. The widths of the blue internal rectangles is w - the distance between the inside edges of the root <g> border - and it is called the containerWidth. That is -

containerWidth = svgNode.width.baseVal.value;

containerHeight = svgNode.height.baseVal.value;

w = containerWidth - (containerBorderWidth * 2);

In other words, the container below is the light-gray border and what is drawn inside it.

Note that bw (border width) below is the width of the borders of the items drawn inside the container. So, this shows that when drawing an item to properly fill a container - and not draw outside the inner bounds of the container - you must consider the border width of the item.
